The Universal Standard for
AI Interaction with Websites

An open standard that allows any website to declare how AI agents can interact with it. Like robots.txt, but for AI.

What is ia.json?

A JSON file at your domain root that tells AI agents exactly what your site can do.

Existing

robots.txt

Tells search crawlers which pages they can index. Text-based, limited to allow/disallow rules.

Emerging

llms.txt

Tells LLMs about site content in Markdown format. Content-focused, no API interaction.

New

ia.json

Tells AI agents what API endpoints exist, how to authenticate, and how to interact safely.

How It Works

Four simple steps from discovery to interaction.

1

Discover

AI agent fetches yoursite.com/ia.json

2

Register

AI registers and gets API credentials (if required)

3

Authenticate

AI signs each request with HMAC signature

4

Interact

AI calls your API endpoints reliably

Simple by Design

Create your ia.json in 5 minutes. Here's a minimal example:

{
  "version": "1.0.0",
  "site": {
    "name": "My Store",
    "type": "ecommerce",
    "currency": "USD"
  },
  "api": {
    "base_url": "https://mystore.com/api/v1",
    "public": {
      "search_products": {
        "method": "GET",
        "path": "/products/search",
        "description": "Search products by keyword",
        "parameters": {
          "q": {
            "type": "string",
            "required": true,
            "description": "Search query"
          }
        }
      }
    }
  }
}

Built for the AI Era

Everything an AI agent needs to interact with your site safely and reliably.

🔍

API Discovery

Declare all your endpoints with methods, parameters, and descriptions.

🔒

Authentication

Built-in support for HMAC signing, OAuth2, API keys, and bearer tokens.

🛡

Access Control

Three access levels: public, AI-authenticated, and user-authorized.

⚡

Rate Limiting

Declare rate limits per endpoint and globally. Auto-block misbehaving agents.

🔔

Webhooks

Declare webhook events so AI agents can receive real-time notifications.

✅

Validation

JSON Schema for validation. CLI tool and online validator included.

Get Started

1. Create your ia.json file

Place a file named ia.json at the root of your domain with your API endpoints.

2. Validate it

npx ia-json-validator https://yoursite.com/ia.json

3. Deploy

That's it! AI agents can now discover and interact with your site.

Read the Quick Start Guide

1. Discover a site's capabilities

import { IaJsonClient } from '@iajson/client';

const client = await IaJsonClient.discover('example.com');

2. Call endpoints

const results = await client.call('search_products', { q: 'laptop' });
