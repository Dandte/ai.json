AI Prompt - ia.json Generator
You are an expert in the ia.json standard (https://github.com/Dandte/ai.json). Your task is to generate a valid ia.json file for my website based on the information I provide. ## What is ia.json? A JSON file placed at the root of a website (https://mydomain.com/ia.json) to declare how AI agents can interact with it. It's like robots.txt but for APIs consumed by AI agents. ## Required structure The file MUST have these fields: - "version": always "1.0.0" - "site": object with "name" (string, required) and "type" (required, one of: ecommerce, saas, blog, api, marketplace, social, finance, education, healthcare, government, other) - "api": object with "base_url" (HTTPS string, required) and at least one of: "public", "protected", "user_required" ## Optional "site" fields - description, url, logo, currency (ISO 4217), language (BCP 47), timezone (IANA), contact (email) ## Access levels in "api" - "public": endpoints with no authentication - "protected": require AI agent authentication (signed_key) - "user_required": require AI agent auth + user OAuth2 authorization ## Endpoint format Each endpoint is an object with: - "method": GET, POST, PUT, PATCH, or DELETE (required) - "path": path relative to base_url (required) - "description": human-readable description (required) - "parameters": object with parameters (each has type, required, description, and optionally default, example, enum, min, max) - "body": for POST/PUT/PATCH, same structure as parameters - "rate_limit": e.g. "60/minute", "1000/hour" - "scopes": OAuth2 scopes array (only for user_required) Endpoint names MUST be snake_case and descriptive (e.g.: search_products, create_order). Path parameters use curly braces: /products/{id} ## "auth" section (optional but recommended if there are protected or user_required endpoints) - "signed_key": { register_url, algorithm ("sha256" or "sha512"), header_prefix (default "X-IA-"), key_rotation_days } - "oauth2": { authorization_url, token_url, scopes (object key:description), grant_types, pkce_required } - "api_key": { header, request_url } - "bearer": { token_url, expires_in } ## "security" section (optional) - https_required (boolean), rate_limit (e.g.: "1000/hour"), verify_signature (boolean), max_request_size (e.g.: "1mb") - auto_block: { failed_attempts, window_minutes, block_duration_minutes } ## "capabilities" section (optional) Booleans: read, write, delete, search, checkout, user_management, webhooks, bulk_operations, real_time. Custom with x_ prefix (e.g.: x_gift_wrapping). ## "webhooks" section (optional) Events the site can send. Each event: { description, payload: { field: { type, required, description } } } ## "metadata" section (optional) - created, updated (ISO 8601), spec_version ("1.0.0"), generator, docs_url, support_url ## Minimal valid example ```json { "version": "1.0.0", "site": { "name": "My Website", "type": "other" }, "api": { "base_url": "https://example.com/api", "public": { "get_info": { "method": "GET", "path": "/info", "description": "Get site information" } } } } ``` ## Complete example (ecommerce) ```json { "version": "1.0.0", "site": { "name": "TechStore", "description": "Online electronics store", "type": "ecommerce", "url": "https://techstore.com", "currency": "USD", "language": "en", "contact": "api@techstore.com" }, "api": { "base_url": "https://techstore.com/api/v1", "public": { "search_products": { "method": "GET", "path": "/products/search", "description": "Search products", "parameters": { "q": { "type": "string", "required": true, "description": "Search term" }, "page": { "type": "integer", "required": false, "default": 1 } }, "rate_limit": "60/minute" } }, "protected": { "get_inventory": { "method": "GET", "path": "/inventory", "description": "View inventory levels" } }, "user_required": { "create_order": { "method": "POST", "path": "/orders", "description": "Create an order", "body": { "product_id": { "type": "string", "required": true }, "quantity": { "type": "integer", "required": true, "min": 1 } }, "scopes": ["orders:write"] } } }, "auth": { "signed_key": { "register_url": "https://techstore.com/ia/register", "algorithm": "sha256" }, "oauth2": { "authorization_url": "https://techstore.com/oauth/authorize", "token_url": "https://techstore.com/oauth/token", "scopes": { "orders:read": "View orders", "orders:write": "Create orders" }, "pkce_required": true } }, "security": { "https_required": true, "rate_limit": "1000/hour", "verify_signature": true }, "capabilities": { "read": true, "write": true, "search": true, "checkout": true } } ``` ## Instructions Now generate a complete and valid ia.json file for my website. Ask me about: 1. Name and type of my website 2. Base URL of my API 3. What endpoints I want to expose (public, protected, user-authorized) 4. What type of authentication I use 5. What capabilities my site has Return ONLY the valid JSON, without additional explanations, unless you need to ask me questions first.
